# spam-patterns.txt — version-controlled regex patterns for known spam.
#
# One pattern per line. Lines starting with '#' are comments. Blank lines
# ignored. Each pattern is a JS regex source (no surrounding slashes,
# case-insensitive matching is applied by the loader).
#
# When a post or comment matches any pattern, plato:
#   1. Sets collapsed_at on the target (auto-hide).
#   2. Attaches a system flag (category='spam', note='pattern: <line>').
#   3. The target appears in /modlog open mode for mod review.
#
# Operator workflow: append a new line per spam wave. Patterns are
# immediate (no restart) if the file is re-read on every write — but the
# default is read-once at boot. Restart to pick up edits, or move to a
# watcher in a future revision.
#
# PRD §Spam Defenses 9. Conservative starter set; broaden over time.

# === Phone-number-with-text-me ===
# The phone shape requires ≥9 actual digits via `\d(?:[\s().-]*\d){8,}`
# (one digit, then 8+ separator-or-digit-then-digit groups). Real phone
# numbers internationally are 8–15 digits; 9 is the safe lower bound.
# Without this, the old `[\d\s().-]{8,}` pattern matched any 10-char
# digit-and-separator string — dates (2026-05-09), version numbers
# (v1.2.3.4.5.6.7), version-shaped tokens — within 30 chars of "telegram"
# or "whatsapp", which is a common shape in posts that legitimately list
# popular messengers in prose.
\b(?:text|whatsapp|telegram).{0,30}\+?\d(?:[\s().-]*\d){8,}\b
